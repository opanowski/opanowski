1. Awal Mula: Kenapa Perlu Dua Device?

Sebenernya ini berawal dari pertanyaan sederhana: "Bro, update website dari tablet bisa ga?"

Gw pikir-pikir, kenapa enggak? Mac gw udah proven — tapi kan ga selalu harus pegang Mac. Kadang lagi santai pegang tablet, ide dateng, pengen langsung update blog. Masa harus nunggu buka Mac dulu?

Apalagi tablet gw bukan tablet biasa. Lenovo Xiaoxin Pad 2024 ini gw install Tiny PC — Debian Linux versi ringan yang jalan di atas Android. Jadi tablet gw technically punya dua OS: Android buat daily use, Linux buat kerja serius kayak coding dan git.

📦 Dua Device, Satu Misi
🍎 MacBook Pro Mid 2015. Device utama bunker. VSCode, Live Server, Terminal — semua udah terkonfigurasi sempurna.
📱 Lenovo Xiaoxin Pad 2024 + Tiny PC. 8GB RAM, 128GB storage, Debian Linux. Tablet yang technically punya dua OS.

Tantangannya: gimana caranya dua device ini bisa update website yang sama tanpa saling nabrak? Jawabannya: Git.

2. Setup Tablet Linux: Dari Nol

Pertama gw harus mastiin tablet siap dulu. Buka Tiny PC, masuk ke QTerminal — ini terminal Linux-nya.

QTerminal — Tiny PC Debian Linux
# Langkah 1: cek git
$ git --version
git version 2.39.5 ← aman, git udah ada
 
# Langkah 2: setup identitas
$ git config --global user.name "opanowski"
$ git config --global user.email "opanowski@gmail.com"
 
# Langkah 3: clone repo (pakai path lengkap, bukan ~)
$ git clone https://github.com/opanowski/opanowski.git \
  /home/tiny/Desktop/opanowski

Nah, next tantangan — clone folder website dari GitHub ke tablet. Di sini drama pertama muncul.

⚠️ Drama #1 — Tanda Tilde (~) Ga Bisa Diketik

Keyboard wireless gw tiba-tiba ga bisa ngetik tanda tilde (~) — tanda yang biasa dipake buat nunjuk home directory. Solusinya? Ganti ~ dengan path lengkap: /home/tiny/. Praktiknya sama persis, cuma lebih eksplisit.

⚠️ Drama #2 — Linux Itu Case-Sensitive!

Udah clone, pas mau masuk foldernya pakai cd, selalu "No such file or directory." Ternyata masalahnya sepele: huruf D di Desktop harus kapital. Linux beda sama Mac yang lebih toleran soal ini.

Case-Sensitive — Linux vs Mac
$ cd /home/tiny/Desktop/opanowski ← BENER
$ cd /home/tiny/desktop/opanowski ← SALAH
 
# Begitu huruf D kapital, langsung masuk. Lega.
$ ls
blog/ images/ latest-posts.js sitemap.xml ... ← Clone sukses!

3. Aturan Main: Bolak-Balik Device

Ini bagian yang paling penting. Bolak-balik device itu boleh, tapi ada satu aturan yang ga boleh dilanggar:

⚡ Aturan Emas Multi-Device
🔒 PULL DULU SEBELUM EDIT — DI DEVICE MANAPUN. Kalau lo edit di Mac, push, terus langsung edit di tablet tanpa pull dulu — tablet lo masih punya versi lama. Git bingung, konflik terjadi.
Alur Kerja Standar — Mac & Tablet
# Berlaku sama persis di Mac maupun tablet
 
$ git pull origin main ← ambil update terbaru
... edit file yang diperlukan ...
$ git add .
$ git commit -m "pesan perubahan"
$ git push origin main
⌨️ Shortcut Keyboard
Mac vs Tablet — Yang Beda Cuma Ini

Mac → Cmd+S (save), Cmd+F (cari), Cmd+K (clear terminal)
Tablet → Ctrl+S (save), Ctrl+F (cari), Ctrl+L (clear terminal)
Panah atas ↑ = ulangi command sebelumnya — berlaku di keduanya.

5. Perbandingan Jujur: Mac vs Tablet

Setelah nyoba dua-duanya, ini perbandingan jujur dari pengalaman gw hari ini:

🍎 Mac — MacBook Pro Mid 2015
+Live Server — preview langsung di browser
+VSCode lebih nyaman di layar besar
+Keyboard fisik full size, lebih enak ngetik
+Semua shortcut udah hafal
−Harus duduk di meja, kurang fleksibel
−Ga bisa dibawa-bawa kalau lagi di luar
🐧 Tablet Linux — Xiaoxin Pad + Tiny PC
+Bisa dibawa kemana-mana
+Update blog sambil rebahan 😄
+Git jalan sempurna, push berhasil
+VSCode tersedia di menu Programming
−Layar kecil, susah edit kode panjang
−Keyboard wireless kadang bermasalah
−Belum bisa Live Server yang proper

Kesimpulan: Mac tetap jadi device utama buat nulis dan ngoding serius. Tablet jadi device backup yang powerful buat update cepat kalau lagi ga pegang Mac.

7. Pelajaran Hari Ini

📋 5 Pelajaran dari Setup Multi-Device
1. Git itu device-agnostic. Selama ada git dan koneksi internet, device apapun bisa jadi "studio" update website lo.
2. Linux itu case-sensitive. Desktop beda sama desktop. Satu huruf kapital bisa bikin error yang bikin garuk-garuk kepala.
3. Tanda ~ bisa diganti path lengkap. /home/username/ = ~/ dalam praktiknya.
4. Pull dulu, baru edit. Ini aturan emas yang ga boleh dilupain kalau pakai banyak device.
5. Keyboard wireless bisa jadi musuh dalam selimut. Beberapa karakter ga mau keluar tiba-tiba — solusinya copy-paste atau cari alternatif path.
