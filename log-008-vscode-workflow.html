Hari ini gw resmi naik level bro. Workflow Lama vs Workflow Baru

❌ Workflow Lama — Terminal Buta
1. Edit file langsung di Terminal pake nano atau cat
2. git add → commit → push ke GitHub
3. Tunggu 1-3 menit GitHub Pages deploy
4. Buka browser cek hasilnya
5. Kalau salah → ulang dari awal 😅
✅ Workflow Baru — VSCode + Live Server
1. Buka folder repo di VSCode
2. Edit file langsung dengan syntax highlighting
3. Klik kanan → Open with Live Server
4. Browser auto-refresh tiap save — preview instan!
5. Kalau udah oke baru git push ke GitHub

Live Server — Game Changer!

Extension satu ini yang paling ngubah cara kerja gw. Live Server bikin website lo jalan di browser lokal di alamat localhost:5500 — tanpa perlu upload ke GitHub dulu. Setiap kali lo save file di VSCode, browser otomatis refresh. Perubahan keliatan dalam hitungan detik!

⚡ Cara Install Live Server di VSCode
1. Buka VSCode
2. Tekan Cmd+Shift+X (buka Extensions)
3. Ketik "Live Server" → Install (dari Ritwick Dey)
4. Buka folder repo: File → Open Folder
5. Klik kanan index.html → Open with Live Server
6. Browser otomatis buka di http://127.0.0.1:5500

Pelajaran Git Hari Ini

Ada beberapa hal soal git yang gw pelajari hari ini — terutama waktu push ke GitHub tiba-tiba kena error merah.

🚨 Drama: Git Push Rejected!

Pas mau push LOG #007, Terminal langsung keluar error merah: "rejected — fetch first". Panik? Iya. Tapi ternyata solusinya simpel — ada perubahan di GitHub yang belum ada di lokal. Tinggal git pull dulu, baru push lagi.

# Kalau kena error "rejected — fetch first": git pull origin main --rebase git push origin main
📚 Rule of Thumb Git
✅ Selalu git pull dulu sebelum mulai kerja
✅ git status buat cek apa aja yang berubah
✅ Commit pesan yang jelas: "Add LOG #008 - ..."
✅ Push hanya setelah tes lokal di Live Server

Jebakan Path Gambar: Lokal vs GitHub Pages

Ini yang paling sering bikin bingung bro. Cara website lo ngakses file gambar itu beda antara di lokal (Live Server) dan di GitHub Pages.

🗺️ Perbedaan Path yang Wajib Dipahami
GitHub Pages: URL-nya /opanowski/images/foto.jpg
Live Server (lokal): URL-nya images/foto.jpg atau ../images/foto.jpg
💡 Jadi gambar yang hilang di lokal belum tentu hilang di GitHub Pages — dan sebaliknya!

Hari ini gw kena dua-duanya: foto hero di homepage ilang di lokal tapi oke di GitHub, terus foto Bayu di LOG #007 oke di lokal tapi ilang di GitHub. Habis ngulik, ketauan masalahnya ada di prefix path ../images/ yang harusnya cukup images/ karena file HTML-nya ada di root folder.

"Tes lokal dulu, push belakangan. Jangan sampe website publik jadi tempat debug." — Bunker Opanowski, 2026

📌 Kesimpulan

Upgrade dari Terminal buta ke VSCode + Live Server itu bukan soal gengsi atau ikut-ikutan. Ini soal efisiensi. Dulu satu perubahan kecil bisa makan waktu 5-10 menit buat liat hasilnya. Sekarang? Detik langsung keliatan.

Workflow baru Bunker Opanowski sekarang: edit di VSCode → preview di Live Server → kalau oke baru push ke GitHub. Simple, aman, dan ga bikin website publik jadi korban eksperimen.

Buat lo yang juga ngurusin website sendiri tanpa background coding — VSCode + Live Server itu investasi waktu 5 menit yang balik modalnya berkali-kali lipat. Coba aja bray!
